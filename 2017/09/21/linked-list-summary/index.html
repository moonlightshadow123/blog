There are a few commonly used techniques in problems related to linked list. Once you master the techniques, most problems are easy to solve.

While loop routines.

  • steps: # generate, # operate, # move
  • variables: …pparentnode, parentnode, curnode, childnode, cchildnode,…

More specifically:

  1. (p)parenetnode and curnode are pre-defined variables and need to be move inside the loop (move it in one line so that you won’t commit any mistakes due to the order of moving);
  2. (c)childnode is generated in the loop, so you don’t have to move it;
  3. curnode (and curnode.next if necessary) is to control the termination of while loop.

You may also use pnode, node, cnode in replace of the above variable names.

Leetcode-237

In this solution, we just copy node.next to node, and then delete node.next.

Consider a situation when head.next = tail(1) head.next != None(2) in self.recur(), which won’t terminate in the current loop. Then in next second recursion, because of low = mid, still there’s still a chance node1.next = tail(1) node1.next != None(2), which indicates a dead loop.
